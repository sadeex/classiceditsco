Classic Creative Co - GitHub Pages Hosting Guide (Firebase Version)

1. Create a new GitHub repository.
   Example: classic-creative-co

2. Upload ALL files inside this folder to the repository root:
   - index.html
   - post.html
   - admin.html
   - assets/
   - .nojekyll

3. In GitHub repository:
   - Open Settings
   - Open Pages
   - Under "Build and deployment"
   - Source = Deploy from a branch
   - Branch = main
   - Folder = / (root)
   - Save

4. Wait 1-3 minutes.

5. Your website URL will be similar to:
   https://YOUR-USERNAME.github.io/REPOSITORY-NAME/

Firebase note:
- This version uses Firebase Authentication, Firestore, and Storage.
- That means posts saved from admin panel can appear on all devices.
- Before publishing, make sure Firebase Authentication, Firestore, and Storage are enabled.
- Read: FIREBASE-SETUP.txt

Important:
- Arena preview may not show live Firebase data because external network access can be blocked.
- On the real hosted GitHub Pages URL, the live Firebase site should work.
